# Design Discussions

This folder contains design documents for features that need discussion before implementation.

Each file is a living document — a conversation between contributors (human and AI) working through the design space of a feature. The goal is to capture the problem, constraints, open questions, and proposed solutions so that by the time we start coding, the design is solid.

## Format

Each document follows a loose structure:

- **Problem**: What are we solving?
- **Constraints**: What must be true? What can't we depend on?
- **Open Questions**: Things we haven't decided yet.
- **Discussion**: The actual back-and-forth — ideas, pushback, refinements.
- **Decision Log**: When a question gets resolved, it moves here with the rationale.

## Rules

- No premature solutions. Understand the problem first.
- Push back on bad ideas. Agreeing to be polite is worse than disagreeing to be right.
- It's okay to leave things unresolved. Mark them and come back.
- Keep it casual. This isn't a spec, it's a conversation.
