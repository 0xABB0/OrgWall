# Probe

This folder is for testing ideas before they become design docs or engine code.

Use probes for:
- API sketches
- architecture experiments
- tradeoff notes
- small example-driven explorations

A probe is allowed to be:
- incomplete
- opinionated
- contradictory with current implementation

A probe is not:
- a stable engine contract
- a guarantee that something will ship unchanged
- a substitute for the design docs under `design/`

Current probes:
- `rendering.frame-recipe.probe.md` — examples of the intended rendering stack from simple 2D to GPU-driven modern paths
